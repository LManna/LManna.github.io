hexo + gitHub 搭建免费博客（有些坑）

  • 随笔
日志

首次踏入github + hexo搭建的博客世界

搭建的过程还是有一些坎坷的，主要原因除了自己太菜还是自己太菜;想想过程还是挺简单的,
按照各路前辈的引路还是能够很快搭建成功的,可惜过程中还是免不了出现各种小错误,报出各种的error！！！

简述搭建的过程(参考hifo)

1.安装软件
(1) 下载安装Git
(2) 下载安装Node.js
(3) 验证是否安装成功
git –version
node -v
npm -v

  1. 注册登录GitHub,配置GitHub
    (1) 一路默然,到最后按默认点击“Finish sign up”
    !(/img/a.png)
    (2) 点击”New repository”，新建一个版本仓库
    !(/img/b.png)
    (3) 输入Repository name:yourname.github.io(这就是你博客的域名了) 
    !(/img/c.png)
!(/img/d.png)
  2. 启用GitHub Page
    (1) 点击右边的“Setting”菜单进入设置,点击”Launch automatic page generator”
    !(/img/3.1.png)
    (2) 点击底部的”Continue to layouts” 
    !(/img/3.2.png)
    (3) 最后点击”Publish page”,发布github默认生成的一个静态站点
    !(/img/3.3.png)
    (4) 验证邮箱 - 点击个人设置 
    !(/img/3.4.png)
    (5) 点击”Send verification Email”发送验证邮件 
    !(/img/3.5.png)
    (6) 进入你的邮箱，查收验证邮件进行验证就可以了
  3. 安装Hexo
    hexo是基于node.js的静态博客，官网也是搭建在GitHub上
    (1) 在电脑上新建一个blog文件夹,该文件夹用于存放你的博客文件,然后右键单击选择“Git Bash”
    (2) 安装hexo,可以使用淘宝的npm镜像,输入一下命令;
    "$ npm install -g cnpm --registry=https://registry.npm.taobao.org"
    (3) 使用淘宝NPM安装Hexo
    "$ cnpm install -g hexo-cli"
 继续输入以下命令
"$ cnpm install hexo --save"
查看是否安装成功"$ hexo -v"
    (4) 本地运行hexo
    $ hexo init
安装生成器
$ cnpm install
    至此执行 $ hexo server 打开浏览器localhost:4000 一路顺畅没有报错的话 应该是可以看到本地的博客
    一般hexo命令
    hexo new”postName” #新建文章
    hexo new page”pageName” #新建页面
    hexo generate #生成静态页面至public目录
    hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
    hexo deploy #将.deploy目录部署到GitHub
    hexo help # 查看帮助
    hexo version #查看Hexo的版本

安装过程中遇见的问题总结

1.主要是在_config.yml配置文件中的最后环节仓库地址：repository后填写http地址还是SSH地址;
一般当填写http对应的仓库地址,每次提交的过程都会输入用户名和地址,所以更多的人选择SSH秘钥的形式;
我最后选择SSH秘钥的形式原因很简单：http地址的格式一直在报错(SSH也在报错,最后还是选择了SSH方式);
简述过程
1.1 配置Git(工作中已经有了一个Git,所以需要在配置出一个专门用来个人使用)
1.ssh-keygen -t rsa -C “new email”
注：首次创建仓库的时候一般我们默认直接敲回车就好，默认生成id_rsa和id_rsa.pub,但是当你本地已经有了一个git仓库,直接敲回车，会覆盖你之前仓库的配置;所以这一步我们需要重新创建第二个文件来存放仓库的秘钥,比如id_rsa_private;
之后他会生成两个文件,id_rsa_private和id_rsa_private.pub;而id_rsa_private.pub里面存放的就是当前邮箱对应仓库的
秘钥地址;
2.配置~/.ssh/config文件;没有的话自己创建一个也行;
‘# 该文件用于配置公司对应的服务器

 # Default github user()
Host github.com
HostName github.com
PubkeyAuthentication yes
PasswordAuthentication yes
User git
IdentityFile /.ssh/id_rsa

# second user()
# 个人github;
Host gitPrivate
HostName github.com
PasswordAuthentication yes
User git
IdentityFile /.ssh/id_rsa_private'
默认走的是公司对应邮箱生成的秘钥的git仓库;
测试配置文件是否正常工作  shh -T -v git@github.com
3.执行ssh-agent让ssh识别新的私钥
ssh-add ~/.ssh/id_rsa_private
该命令如果报错：Could not open a connection to your authentication agent.无法连接到ssh agent，可执行ssh-agent bash命令后再执行ssh-add命令。
